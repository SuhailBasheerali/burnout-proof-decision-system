### Zone 2: TIME_BOX â±ï¸
**Growth â‰¥70 AND Sustainability <50**

```

**Recommendation**:
```

1. Announce sprint duration explicitly ("90-day push")
2. Mobilize team for intensive effort
3. Relax secondary priorities
4. Plan celebration at finish line

```

**Recovery Plan**:
```
During Recovery (Post-Sprint):
- Reduce new commitment rate by 50%
- Focus on technical debt reduction
- Rebuild team morale & relationships
- Process documentation & standardization
- Mental health check-ins
- 2-3 week vacation time for key people
```

**Time-Box Duration Rules**:
```
Sprint Duration â†’ Recovery Duration â†’ Safe Repeat?
2 weeks sprint â†’ 1 week recovery â†’ Every week OK
4 weeks sprint â†’ 2 weeks recovery â†’ Monthly OK
12 weeks sprint â†’ 4-6 weeks recovery â†’ Quarterly OK

**Signs You're Doing It Right**:
- Team knows the end date
- There's a visible recovery phase planned
- Metrics show stress, but within limits
- People aren't working unsustainable hours (>60 hr/week) repeatedly

**Red Flags**:
- Team has lost sight of end date
- No recovery phase in sight
- Working 70+ hours multiple weeks in a row
- Attrition starts showing up
- Decision maker says "Oh, this will just be 2 more weeks..." for the 5th time

## Risk Level System

Risk levels identify **what could go wrong** and the severity.

### STRUCTURALLY_STABLE âœ…

**Meaning**: Balanced decision, low burnout risk

**Characteristics**:
- Growth â‰¥ 60 and Sustainability â‰¥ 60
- Tension index is LOW or MODERATE
- Team has capacity
- Risk factors are minimal

**What to Do**:
- Proceed with confidence
- Focus on execution quality
- Monitor for deviations, but not critical

**Example**:
```

Risk: STRUCTURALLY_STABLE âœ…
â†’ Healthy, sustainable decision
```

---

### SEVERE_BURNOUT_RISK âš ï¸ CRITICAL

**Meaning**: High growth demands exceed sustainability capacity

**Characteristics**:
- Growth â‰¥ 75 AND Sustainability < 35, OR
- Growth â‰¥ 70 AND Sustainability < 50 AND Tension CRITICAL

**What to Do**:
1. **IF ALREADY COMMITTED**: Time-box it
   - Set explicit end date (90 days max)
   - Plan recovery period (4-6 weeks mandatory)
   - Do not enter recovery into "business as usual"
   
2. **IF NOT COMMITTED YET**: Redesign
   - Reduce growth targets, OR
   - Invest in sustainability first, OR
   - Find different approach

**Burnout Trap Example**:
```

Tension: 61 (critical imbalance)
Risk: SEVERE_BURNOUT_RISK âš ï¸

Composite Score: 43.73/100 (heavily penalized)

Recommendation: Time-box for 12 weeks, then mandatory 6-week recovery
```

---

### SUSTAINABILITY_DEFICIT âš ï¸

**Meaning**: Team doesn't have capacity to execute

**Characteristics**:
- Sustainability score < 40
- Usually accompanied by moderate-to-high growth targets
- Team is at/near breaking point

**What to Do**:
1. **Increase capacity first**: 
   - Hire support
   - Delegate/outsource
   - Upgrade tools/process
   - Reduce scope
   
2. **Then execute**:
   - With improved sustainability, re-evaluate
   
3. **Or reduce growth targets**:
   - What's actually achievable at current capacity?

**Example**:
```
Growth: 60, Sustainability: 25
Risk: SUSTAINABILITY_DEFICIT âš ï¸

Analysis: Team is overextended
Action: Hire 2 engineers first, then re-evaluate
```

---

### GROWTH_STAGNATION_RISK âš ï¸

**Meaning**: Insufficient progress toward goals

**Characteristics**:
- Growth score < 40 (except in LIGHT_RECOVERY zone)
- Initiative not ambitious enough
- Value creation is marginal

**What to Do**:
1. **Increase ambition**:
   - What would make this 2x more impactful?
   - Can scope be expanded?
   - Are there adjacent opportunities?
   
2. **Or deprioritize**:
   - Not worth doing if value is low
   - Free up capacity for high-impact work

**Example**:
```
Growth: 35, Sustainability: 60
Risk: GROWTH_STAGNATION_RISK

Current state: Safe but low-value
Question: Are there related initiatives with 2x impact?
Action: Either redesign for higher impact, or stop
```

---

### LOW_STRUCTURAL_VALUE âš ï¸

**Meaning**: Weak in both dimensions

**Characteristics**:
- Growth < 40 AND Sustainability < 40
- Neither growth nor sustainability adequate
- Should be AVOIDED entirely

**What to Do**:
- Don't do it
- Redesign fundamentally, OR
- Find alternative approach
- If forced to proceed: explicitly understand trade-offs

---

### SEVERE_IMBALANCE (Internal Risk)

**Meaning**: Lopsided approach (not primary classification, but noted)

**Characteristics**:
- Tension index > 50
- One dimension much stronger than other
- Unstable long-term

**Mitigation**:
- If high-growth, low-sustainability: Use time-box approach
- If low-growth, high-sustainability: Good recovery mode
- Aim to rebalance over time

## Stability Classification

### Purpose
Measures how **robust** a decision is to changes in criteria.

### How It Works
System perturbs each criterion weight by Â±20% and measures variance in composite score.

```
If perturbing weights slightly changes the decision â†’ FRAGILE
If decision remains stable â†’ STABLE
```

### Interpretation

| Stability | Variance | Robustness | Action |
|-----------|----------|-----------|--------|
| **STABLE** | <8 | Very robust | Trust the decision |
| **MODERATELY_STABLE** | 8-20 | Fairly robust | Track key metrics |
| **FRAGILE** | >20 | Low robustness | Verify assumptions |

### STABLE ğŸŸ¢
```
Example:
- G: 75, S: 75, Sensitivity: 2.3
- Even if growth weights are wrong by 20%, decision holds
- Even if sustainability weights change, conclusion same

Interpretation:
- Assessment is solid
- Criteria weighting is reasonable
- Safe to proceed with confidence
- Focus on execution, not re-evaluation
```

### MODERATELY_STABLE ğŸŸ¡
```
Example:
- G: 75, S: 50, Sensitivity: 14.5
- If growth weights are high, stronger feeling to proceed
- If growth weights are low, more cautious

Interpretation:
- Assessment is reasonable, but assumptions matter
- Verify key metrics during execution
- Be ready to adjust if metrics diverge
- Monitor top 2-3 criteria closely
```

### FRAGILE ğŸ”´
```

- G: 55, S: 48, Sensitivity: 23.1
- Small changes in weight assumptions changes decision
- Marginal calls are unreliable

Interpretation:
- Decision is borderline
- Assumptions are tight
- Should NOT force this decision yet
- Either: clarify criteria OR reconsider approach
- If proceeding anyway, close monitoring required
```

## Framework Maturity Levels

### Level 1: Reactive
- Decisions made by gut feel
- No systematic evaluation
- Burnout is a surprise

### Level 2: Growth-Focused (Initial)
- Using framework for growth assessment
- Still ignoring sustainability
- Burnout still common

### Level 3: Balanced (Mature)
- Using framework to balance growth and sustainability
- Making time-box + recovery decisions consciously
- Burnout is exceptional, planned-for event

### Level 4: Predictive (Advanced)
- Predicting burnout risks based on patterns
- Adjusting organization structure proactively
- No unplanned burnout

### Level 5: Optimized (Excellence)
- Running on sustainable high-velocity
- Growth and sustainability both consistently high
- Time-boxing rare, recovery periods planned

---

This framework transforms burnout from "something that happens" to "something we prevent."
